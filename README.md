# ObjPxlLiveDiagnostics

Monorepo for ObjectivePixel's live diagnostics tooling, containing a telemetry client library and a companion viewer app.

## Components

### ObjPxlLiveTelemetry (Client Library)

Lightweight, multi-platform telemetry client for ObjectivePixel apps. Targets iOS, macOS, visionOS, tvOS, and watchOS.

**Location:** `src/DiagnosticClient/`

#### Installation

In Xcode: **File > Add Packages...** and use the repository URL.

With a `Package.swift`:

```swift
dependencies: [
    .package(url: "https://github.com/ObjectivePixel/ObjPxlLiveDiagnostics.git", from: "0.2.2")
]
```

Then add `ObjPxlLiveTelemetry` to your target's dependencies:

```swift
.target(
    name: "YourTarget",
    dependencies: [
        .product(name: "ObjPxlLiveTelemetry", package: "ObjPxlLiveDiagnostics")
    ]
)
```

#### Quick Start

```swift
import ObjPxlLiveTelemetry
import SwiftUI

@main
struct MyApp: App {
    private let telemetryLifecycle = TelemetryLifecycleService(
        configuration: .init(
            containerIdentifier: "iCloud.com.yourcompany.yourapp.telemetry"
        )
    )

    var body: some Scene {
        WindowGroup {
            NavigationStack {
                TelemetryToggleView(lifecycle: telemetryLifecycle)
            }
        }
    }
}
```

### LiveDiagnosticsViewer (Viewer App)

macOS/iOS app for viewing live diagnostics generated by the ObjPxlLiveTelemetry client.

**Location:** `src/LiveDiagnosticsViewer/`

Open the project in Xcode:

```bash
open src/LiveDiagnosticsViewer/LiveDiagnosticsViewer.xcodeproj
```

## Building

### Client Library

```bash
cd src/DiagnosticClient
swift build
swift test
```

### Viewer App

Open `src/LiveDiagnosticsViewer/LiveDiagnosticsViewer.xcodeproj` in Xcode and build/run from there.

## Platforms

- iOS 17+ / macOS 14+ / tvOS 17+ / visionOS 1+ / watchOS 10+ (Client Library)
- iOS 26+ / macOS 26+ / visionOS 26+ (Viewer App)
